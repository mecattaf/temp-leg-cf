## 1. Executive summary

The big news since Tom's last touchpoint with Cloudflare's roadmap is **Agents Week 2026 (April 13–17)** — Cloudflare spent five days shipping exactly the primitives Leger needs, and several of them reshape the per-surface verdicts. The headline takeaway: Leger's existing 8-surface footprint already qualifies as "meaningfully uses Workers" for Launchpad selection, and a disciplined adoption of ~6–8 more surfaces — Workers AI, AI Gateway, Durable Objects, Secrets Store, Turnstile, Analytics Engine, Workflows (post-sprint), Browser Run (post-sprint) — pushes Leger into the 14–16 product range that characterises Cohort-alumni winners like Langbase and ELYD without violating a single Leger doctrine.

**Enterprise MCP is a complement, not a substitute.** Cloudflare's "enterprise MCP" reference architecture (blog.cloudflare.com/enterprise-mcp, April 2026) stacks Code Mode + MCP Server Portals + AI Gateway + Access + Gateway/DLP on top of per-service MCP Workers — exactly the pattern Leger already has for `sodimo/mcp`. Leger's single-MCP-Worker-per-engagement *is* the Code-Mode-equivalent for that customer's back-office; the governance layer is LP-XX, not MCP Server Portals (which require a Cloudflare One subscription). MCP Server Portals is Enterprise-gated in open beta and out of reach on the Pro tier (zero-trust-mcp-server-portals, December 2025).

**Workers AI + AI Gateway together replace LiteLLM-in-the-cloud-escalation-path, preserving Path (c).** AI Gateway's Universal Endpoint + OpenAI-compatible `/chat/completions` route to Workers AI, Anthropic, OpenAI, Google, and "14+ providers" under one cf-aig-authorization token (ai-gateway/usage/universal, January 2026 and usage/chat-completion, March 2026). The on-box LiteLLM quadlet stays primary (LP-07); AI Gateway becomes the *cloud-escalation shim* — called only when a local-heavy model is unavailable. Token accounting drops into `run_ledger` via cf-aig-metadata headers. No long-lived vendor creds in the configurator — AI Gateway accepts per-request tokens.

**Durable Objects SQLite is now GA and on the free tier** (durable-objects release notes, March 2026), making it the correct home for ephemeral configurator state (OAuth device-grant, GH App install-ID rotation) currently sitting in D1. **Workflows went GA** (workflows-ga-production-ready-durable-execution, April 2025) and has since been re-architected for agentic workloads (workflows-v2, April 2026) — complementary to `leger rotate`/`leger bootstrap` runbooks only for the **vendor-side orchestration** (manifest promotion, release-please fan-out); the on-box CLI verbs stay on-box because the box must survive `app.leger.run` going dark (survivability matrix in §B of `99-consolidated.md`).

**Secrets Store is in open beta and is the right home for the GitHub App private key** (secrets-store-beta, April 2025 and secrets-store/integrations/workers, March 2026). Account-level secrets with binding-based access, propagated via Quicksilver, audit logs — directly addresses "GH App private key rotation" in LP-03 as applied to Leger's own creds. Adoption is sprint-adjacent (Block 5 or Block 6 — literally minutes of `wrangler secrets-store`).

**Turnstile, Analytics Engine, Images, Stream, Browser Run, Workers for Platforms — mixed verdicts.** Turnstile is a trivial `adopt now` for `leger.run` signup gating (turnstile-ga, September 2023) — free, WCAG 2.1 AA compliant, two lines of HTML. Analytics Engine is the correct home for Leger-Labs-wide aggregate `tokens-local` / `cost-saved` telemetry (analytics/analytics-engine) — unlimited-cardinality blobs, SQL API, free tier. Images and Stream are `reject` (wrong problem — R2 + Caddy already covers per-customer assets, and Leger does not ship video artifacts as a core deliverable). Browser Run (formerly Browser Rendering) is `adopt post-sprint` for release-artifact screencasts and docs-site screenshot generation but not sprint-critical. Workers for Platforms is `reject` — dispatch-namespaces are the right pattern for multi-tenant SaaS platforms hosting customer code, which Leger explicitly is not (`00-scope.md` §3 "NOT a multi-tenant SaaS").

**Hyperdrive, Calls/RealtimeKit, Vectorize, Containers, WARP/Browser Isolation/DLP — all reject or evaluate later.** Hyperdrive is for Postgres/MySQL aggregation — irrelevant unless Leger opts into the cross-engagement D1-mirror dashboard (§D.16 below). RealtimeKit is closed beta + overkill (`99-consolidated.md` §noon-ecosystem). Vectorize is worth an `evaluate later` if Blueprint-3 (insurance finetune) ever needs edge RAG. Containers went GA April 13, 2026 (containers-sandbox-ga) but duplicates what the Framework Desktop already does — Leger's value proposition is *not running containers in Cloudflare's network*. WARP + Browser Isolation + DLP are Zero Trust add-ons that only make sense under a Contract plan — blocked by Leger's Pro tier ceiling.

**Pages Functions vs Workers** — Cloudflare is actively migrating Pages projects into Workers-with-static-assets (cloudflare pages vs workers 2026, cogley.jp April 2026). New Leger projects should target **Workers + static assets**, not Pages — `leger-docs`, `leger.run/changelog`, `<customer>.leger.run/changelog` should all use the Workers pattern going forward. This is a documentation-and-configuration change, not a product decision.

**Bundle-deal posture is real but not advertised.** Workers Launchpad's higher credit tiers ($100K / $250K) effectively require VC-backed status, but the base tier plus Workers Paid is ample at Leger's scale. Stacked-surface count does move the needle for Launchpad selection per the ELYD-founder interview paraphrased in `noon-ecosystem-support-map.md`: "the newer and fresher/beta-er the better." Leger at 15 products (current 8 + 7 new) is credibly in that zone.

**The Built on Cloudflare / Small App Garden showcases are submission-gated, not selection-gated.** Both are populated by "get in touch" forms (built-with-workers-jamstack, 2020 and garden.cloudflare.dev); the bar is "you're actually using the platform and have a public landing page." Sodimo first production + `leger.run` live is sufficient.

**The one surface Tom needs to think hardest about is Cloudflare's internal AI engineering stack (`internal-ai-engineering-stack`, April 2026).** Cloudflare *itself* runs "a proxy Worker, Cloudflare Access, AI Gateway, and a client-accessible discovery endpoint" — four of the eight surfaces Leger already uses, configured to give OpenCode users anonymous UUIDs in cf-aig-metadata and per-user cost tracking (internal-ai-engineering-stack). This is a free blueprint for the "Claude Code optional, never load-bearing" Path (c) architecture Leger already committed to, and the Launchpad application should name it explicitly.

- 8 surfaces already in use → 0 new sprint work
- 6 `adopt now` surfaces (rows 1–6) → sprint-adjacent, ≤4h each
